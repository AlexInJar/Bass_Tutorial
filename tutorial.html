2. Build a Stochastic Reward Net with SPNP

To describe and solve for SRN, we must first have a CSPL file, which is a *.c file descrbing how a SRN is constructed.

In general, the CSPL are based on standard ANSI C language file with five predetermined functions:

Function Task Implementation Details
void options(void) Set the way of solving and describing SRN functions iopt() and fopt() respectively set the integer option and float option
void net(void) Build the SRN model using places and transitions - void place(char *p) defines  a place with name p 

-  void init(char *p, int n) defines the initial number of token n in place p (By default, places are empty) 

- void imm(char *t) defines an immediate transition, void probval(char *t, double val) defined the probability mass for that immediate transition 

- void rateval(char *t, double val) defines a timed transition with rate val 

- void priority(char *t, int prio) 
-void policy(char *t, int pol)

- void iarc(char *t, char *p) connect place p to transition t using input arc; void oarc(char *t, char *p) connect transition tto place p using output arc; miarc and moarc defines the multiplicity of that arc

- void halting_condition(int (*gfunction)()) defines the halting function for SPN

(See the manual for defining the net with rate-dependent function, marking-dependent function, fluid stochatic petri net and many more extensions)
int assert(void) Checks whether a marking of the net is legal or not - returns RES_ERROR if marking illegal
- returns RES_NOERR if marking legal
void ac_init(void);void ac_reach(void) Logging the information of the net at the beginning or the end of reachability graph generation - void pr_net_info(void) is called in beginng to output model information to *.out file
- void pr_rg_info(void) is called at the end to output reachability graph information to *.rg file
void ac_final(void) Define the outputs of SPNP. We can print/log our desired metric here. - void pr_mc_info(void) outputs staitionary distribution of markov chain
- void pr_std_average(void) 
- void pr_expected(char *string, double(*func)() a marking-dependent reward function that prints the reward

A simple applicable example of CSPL:

Consider the following petri net model, Figure 4.2 from the SPNP manual

Here is the example of building the said model using CSPLwe define the two times transition with rates 12,21 as the following:

1221=7.3P1=1
/* Global variables */
# include    "user.h" // 
int N; // Global variable 
void options() {
    iopt(IOP_SSMETHOD,VAL_SSSOR); // Solving by Steady-State SOR (Successive Overrelaxation) 
    iopt(IOP_PR_FULL_MARK,VAL_YES); // Print Mark in full format
    iopt(IOP_PR_MARK_ORDER,VAL_CANONIC); /* order when printing the reach set: VAL_CANONIC,VAL_LEXICAL,VAL_MATRIX */
    iopt(IOP_PR_MC_ORDER,VAL_TOFROM); /* order when printing the Markov Chain: VAL_FROMTO,VAL_TOFROM */
    iopt(IOP_PR_MC,VAL_YES); /* how to print the Markov Chain: VAL_YES,VAL_NO */
    iopt(IOP_MC,VAL_CTMC); /* print the DTMC state probs (used if IOP_MC==VAL_DTMC): VAL_YES,VAL_NO */
    iopt(IOP_PR_PROB,VAL_YES); /* print the tangible marking probas: VAL_YES,VAL_NO */
    iopt(IOP_PR_RSET,VAL_YES); /* print the reach set: VAL_YES,VAL_NO,VAL_TAN */
    iopt(IOP_PR_RGRAPH,VAL_YES); /* print the reach graph: VAL_YES,VAL_NO */
    iopt(IOP_ITERATIONS,2000); /* print the Markov chain accumulated time spent in states: VAL_YES,VAL_NO */
    iopt(IOP_CUMULATIVE,VAL_NO); /* print the Markov chain accumulated time spent in states: VAL_YES,VAL_NO */
    fopt(FOP_ABS_RET_M0,0.0); /* rate from absorbing markings to initial marking, if positive */
    fopt(FOP_PRECISION,0.00000001);
    N = input("Please enter the token number N:");
}
void net() { // Note we can also use for loop to construct the markov chain
    place("p_1");
    place("p_2");
    trans("t_12");
    trans("t_21");
    init("p_1",N);
    /* timed transition */
    ratedep("t_12",7.3, "p_1"); // Here the rate depend on the p_1, note that it is different than depend on N
    rateval("t_21",1.0);
    iarc("t_12","p_1");
    iarc("t_21","p_2");
    oarc("t_12","p_2");
    oarc("t_21","p_1");
}
int assert() {
    return RES_NOERR;
}
void ac_init() {
}
void ac_reach() {
}
/*reward rate */
double tokenNo() {
    return(mark("p_1"));
}
double rd() {
    return ((mark("p_2")==0) ? 1:0);
}
void ac_final() {
    solve(INFINITY);
    pr_expected("Average Token Number in place p_1:", tokenNo);
    pr_expected("steady state prob. that place p_2 is empty: ", rd);
}

3. Interpreting the results

After running:

spnp <filename-without-*.c-extension>

We see that some results are being generated:

*.rg contains information on the reachability graph. The reachability graph contains the information on how the system is evolving.

In the simple example, when N=4 in place p1, we have the reachability graph:

Well, let us first inspect the file manual.rg:

Let us focus on the last two blocks:

The first block index the markings we just saw into 0,,4:

#               p_1      p_2      
     0_t        4        :
     1_t        3        1
     2_t        2        2
     3_t        1        3
     4_t        :        4

The second block gives out the reachibility graphs in the form of an adjacency list.

In another words, each row represents how one state is connected to another state through the firing of one transition at certain rate.

For example the third row starting with number 2, gives us the information on how the marking (2,2) can be transitioned to (3,1) or (1,3) though the firing of t_21 and t_12, with rates 1 and 14.6 

_reachgraph =

     1     0:t_21:1.000000e+000     2:t_12:2.190000e+001;

     3     2:t_21:1.000000e+000     4:t_12:7.300000e+000;
